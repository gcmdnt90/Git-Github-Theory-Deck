Module 3

The everyday loop

Five commands cover 90% of daily Git. Learn these and you can work for real.

Be honest: this is the module people actually need. Everything else is occasional. Drill the loop.

The daily loop is five moves

You repeat this cycle all day: see what changed, stage it, save it, share it.

git statusWhat changed and what’s staged. Run it constantly — it’s free.
git add <file>Put a change in the staging tray (git add . for everything).
git commit -m "message"Save a snapshot of the staged changes, with a note.
git pushSend your commits up to GitHub.
git pullBring down commits others pushed. Do this before you start.
status → add → commit → push. Then pull to stay in sync. That’s the job.
Two habits to preach: run status often, and pull before you begin a work session.

Saving work is stage, commit, push

You (human) 
git add .
git commit -m "Add intro section"
git push

Three commands: stage everything, snapshot it with a message, send it to GitHub.

Ask the agent

“Commit the changes with a sensible message and push.”

The agent runs the same three commands. It usually writes the commit message for you — read it before approving.

GUI equivalent: VS Code’s Source Control panel and GitHub Desktop do stage / commit / push with buttons — same three steps.

Point out the message is the one thing worth reviewing on the agent side. Everything else is mechanical.

Review before you save

You (human) 
git status      # what changed
git diff        # the exact line edits
git log --oneline   # history so far
Ask the agent

“What’s changed since my last commit?”

“Summarise the last five commits.”

The agent reads the same output and explains it in plain language.

status = what, diff = which lines, log = history. These three answer “what’s happening?”
diff is the reviewing tool. It's how you check both your own and the agent's changes before committing.

Start once, then loop forever

New project on your machine:

git init            # make this folder a repo
git add .
git commit -m "First commit"

Existing project from GitHub:

git clone https://github.com/user/project.git

Clone downloads the repo and its full history, ready to work.

Or just tell the agent: “Initialise a git repo here” / “Clone this GitHub repo for me.”

init vs clone is the one-time setup. After that it's the daily loop forever.
