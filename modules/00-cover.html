A practical module — you and your agent

Git & GitHub

The smallest set of ideas you need to track your work, undo mistakes, and collaborate — whether you type the commands yourself or ask an AI agent to do it.

Basic literacy module | For humans and their agents | 8 short modules

Frame the promise: by the end you can use git safely for real work. Every action is shown two ways — what you type as a human, and what you say to an agent. The goal is confidence, not completeness.

How this module reads

Git is the same underneath no matter who drives. Throughout the deck, every action appears in two panels:

You (human) 
git commit -m "Fix typo"

You run the command in a terminal (or click the button in VS Code / GitHub Desktop).

Ask the agent

“Commit this with a short message.”

The agent runs the same git command for you — you review and approve the result.

Same git. Two ways to drive it. You should understand both, because you stay responsible either way.
Emphasise: delegating to an agent does not remove your responsibility. You still need the mental model so you can tell whether the agent did the right thing.

One sentence to carry through

Git saves snapshots of your work so you can go back, branch off, and share — and an agent can do all of it for you, but you review before it counts.

Everything else in this deck is detail hanging off that sentence.

This is the thesis. Snapshots + branching + sharing, with a human in the loop. If someone remembers only this slide, they have the core.
