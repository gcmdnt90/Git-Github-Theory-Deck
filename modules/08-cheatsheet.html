Ten commands cover daily Git

Daily loop

git statuswhat changed
git add .stage everything
git commit -m "..."save a snapshot
git pushsend to GitHub
git pullget others’ work

Look & branch & undo

git diffwhich lines changed
git log --onelinehistory
git switch -c namenew branch
git merge namebring branch in
git revert <id>undo a pushed commit
Ten commands. This really is enough for the vast majority of daily work.

The rules fit in one breath

Do: commit small & often · write honest messages · pull before you start · read the diff before you accept · use .gitignore.

Don’t: commit secrets · force-push shared branches · let an agent’s change land on main unreviewed · panic — almost nothing is truly lost.

Snapshots you can trust, branches to experiment, a remote to share — and a human who reviews before it counts.
The do/don't pairing is the safety summary. Close on the thesis sentence one more time.

You bring judgement; the agent brings speed

Your job

Understand the model. Decide what to change. Review the diff. Own what lands.

The agent’s job

Run the commands. Draft messages. Explain history. Propose fixes and conflict resolutions.

You bring judgement; the agent brings speed. Git is the shared, reviewable record between you.
Final framing. This division of labour is the whole point of the module.
