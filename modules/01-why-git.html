Module 1

Why Git is worth it

Before the commands: what problem does it actually solve, and why do people who use it never go back?

Motivation first. If people feel the pain git removes, the commands later feel worth learning.

File-name versioning breaks silently

You know this file folder:

report_final.docx
report_final_v2.docx
report_final_REAL.docx
report_final_use-this-one.docx

It breaks down the moment you need to answer: what changed, when, and can I get the old version back?

And with two people editing, it gets worse: whose copy is right? Which changes got lost when you emailed versions back and forth?

Manual versioning fails silently. You only notice when something is already lost.
Everyone recognises the "_final_final" pattern. This is the emotional hook.

Git makes change recoverable

A time machine

Every saved snapshot is recoverable. Broke something an hour ago? Go back to exactly how it was.

A safety net

Experiment freely. If it goes wrong, you throw the change away — the last good version is untouched.

A team workflow

Many people change the same project at once, and Git merges the work and flags real conflicts.

Git turns “don’t touch it, it works” into “try anything, we can always go back.”
Three benefits: recover, experiment safely, collaborate. Confidence to change things is the real payoff.

Agents make version control non-optional

An AI agent can change dozens of files in seconds. That is powerful — and risky if you can’t see or undo what it did.

Git is what makes agent help safe to accept: you can inspect the diff, save the good work as a commit, and throw away the rest.

With Git: “Show me exactly what you changed” and “undo that” are one command away.
Without Git: an agent’s edits can overwrite files with no reliable record of what changed.
Git is the seatbelt for working with agents. It doesn’t slow you down — it lets you go faster safely.
Tie motivation to the deck's theme: agents make version control non-optional, because they change a lot fast.
